## 需求 2：多模型接入（OpenAI SDK 标准）

### 2.1 支持模型

| 模型 | 渠道 | 特点 |
|------|------|------|
| **Qwen3.5-VL** | 阿里百炼 | 原生多模态，视频理解强 |
| **Doubao-Seed-2.0** | 火山引擎 | 长视频优化，独立 Key |

### 2.2 接口标准化

所有模型统一使用 **OpenAI SDK 兼容格式**：

- 统一的 `/chat/completions` 端点
- 统一的请求/响应格式
- 统一的流式输出处理

### 2.3 API Key 配置

| 渠道 | 环境变量 | 获取地址 |
|------|----------|----------|
| 阿里百炼 | `VITE_DASHSCOPE_API_KEY` | dashscope.console.aliyun.com |
| 火山引擎 | `VITE_ARK_API_KEY` | console.volcengine.com/ark |

### 2.4 验收标准

- [ ] 模型下拉按渠道分组
- [ ] 根据模型自动使用对应 Key
- [ ] 流式输出正常
- [ ] OpenAI SDK 兼容格式

## 需求 3：账号体系

### 3.1 技术方案

采用 **Supabase Auth**：

- 完整认证方案（邮箱密码 + OAuth）
- 行级安全（RLS）保护数据
- 免费额度充足（50,000 MAU）
- 与 Vercel 部署无缝集成

详细设计见 [auth.md](./auth.md)

### 3.2 功能清单

| 功能 | 描述 |
|------|------|
| 注册/登录 | 邮箱密码 + GitHub OAuth |
| 个人中心 | 昵称、头像、邮箱管理 |
| 脚本收藏 | 保存/查看/删除收藏的脚本 |

## 需求 4：脚本收藏

### 4.1 基础功能

- 收藏当前脚本（标题 + 描述）
- 查看收藏列表
- 加载收藏的脚本继续编辑
- 删除收藏

### 4.2 收藏脚本引用（参考生成）

在「参考生成」模式中，可直接从收藏列表选择脚本作为参考源：

```
参考脚本来源：
├── 当前脚本（已完成分析/生成的）
└── 收藏列表（从已收藏的脚本中选择）
```

### 4.3 数据结构

| 字段 | 类型 | 描述 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 用户 ID |
| title | string | 标题 |
| description | string | 描述 |
| script_data | JSON | 完整脚本数据 |
| source_type | string | 来源类型 |
| created_at | timestamp | 创建时间 |

### 4.3 验收标准

- [ ] 一键收藏当前脚本
- [ ] 收藏列表分页展示
- [ ] 加载收藏到编辑器
- [ ] 删除收藏

## 需求 6：多输入源生成

### 6.1 输入模式

| 模式 | 输入 | 用例 |
|------|------|------|
| 视频分析 | 视频文件 | 分析现有视频 |
| 从零生成 | 图片 + 文字 | 创意脚本 |
| 参考生成 | 已有脚本 + 指令 | 调整优化 |

### 6.2 分析结果跳转生成

视频分析完成后，支持一键跳转继续创作：

```
分析完成
    │
    ├── [基于此生成新脚本] → 参考生成模式（自动填充参考脚本）
    │
    └── [继续优化] → 从零生成模式（可补充图片/调整描述）
```

### 6.3 验收标准

- [ ] 三种模式切换
- [ ] 图片上传支持
- [ ] 参考脚本加载
- [ ] 收藏脚本作为参考源
- [ ] 分析结果跳转继续生成
