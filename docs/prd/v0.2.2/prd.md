## 需求 4：脚本收藏

### 4.1 基础功能

- 收藏当前脚本（标题 + 描述）

### 4.2 收藏脚本引用（统一输入）

## 需求 6：多输入源生成

### 6.1 统一输入结构（无 Tab）

脚本生成采用单一表单，不再区分「从零创作 / 参考生成」Tab。  
系统根据用户是否填写参考脚本、参考图片等信息自动决定生成策略。

输入项：

- 视频要求卡片（必填）：视频类型 + 动态字段 + 补充说明
- 参考图片（可选）
- 参考脚本（可选，支持手动输入 / 从收藏选择）
- 风格、时长、脚本数量（可选，其中脚本数量支持 1~N）

### 6.2 视频要求卡片规范

「创作主题」升级为「视频要求」卡片，先选视频类型，再填写对应动态字段。

| 视频类型（7 选 1） | 动态字段（示例） |
|------|------|
| 电商头图类 | 商品名称、核心卖点、价格/优惠信息、目标人群 |
| 使用场景类 | 典型场景、用户痛点、使用步骤、场景道具/环境限制 |
| 卖点展示类 | 卖点 1/2/3、证明方式（数据/对比）、优先级 |
| 开箱测评类 | 开箱亮点、测评维度、优缺点、结论倾向 |
| 对比种草类 | 对比对象、对比维度、推荐理由、适用人群 |
| 教程讲解类 | 教程目标、步骤拆解、注意事项、成果展示方式 |
| 品牌故事类 | 品牌定位、故事主线、情感关键词、行动号召 |

### 6.3 三步流程

```
Step 1 生成脚本
  - 提交统一输入表单
  - 生成固定信息卡片（挂起展示）
  - 同时生成多个脚本候选并支持切换查看

Step 2 生成分镜图
  - 基于当前选中脚本候选
  - 支持批量生成 + 单条生成/重试

Step 3 生成视频
  - 基于“当前选中脚本候选 + 已生成分镜图”
  - 调用视频生成 API 产出最终视频
```

### 6.4 分析结果跳转生成

视频分析完成后，支持一键跳转继续创作，并将分析结果作为参考脚本预填到统一输入表单（用户可继续修改）。

### 6.5 验收标准

- [ ] 脚本生成区无从零/参考 Tab 切换
- [ ] 视频要求卡片支持 7 类视频类型与动态字段
- [ ] 补充说明字段在所有视频类型下可用
- [ ] Step1 提交后固定信息卡片持续展示
- [ ] Step1 支持一次生成多个脚本候选并切换展示
- [ ] Step2 支持分镜图批量生成与单条生成
- [ ] 分析结果可跳转并预填参考脚本

## 需求 7：分镜图与视频生成

### 7.1 技术方案

- 模型：通义万相 `wanx2.1-t2i-turbo`
- 分镜图调用：异步任务 + 轮询
- 视频生成调用：根据选中方案（脚本 + 分镜图）提交视频生成 API
- 分镜图并发：最多 2 个任务

### 7.2 验收标准

- [ ] 批量生成分镜图
- [ ] 单个重新生成
- [ ] 图片展示在表格
- [ ] 可基于当前选中方案触发视频生成
- [ ] 视频生成状态可见（待开始/生成中/已完成/失败）
