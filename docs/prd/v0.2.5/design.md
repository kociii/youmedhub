# v0.2.5 界面设计文档

> 增长与国际化能力增强 - 基于当前三栏工作台与侧边菜单结构增量演进

---

## 1. 当前实现审查结论

### 1.1 当前壳层结构

当前应用并不是“顶部导航 + 内容页”的结构，而是 `AppLayout` 提供的三段式桌面壳层：

```
┌────────────┬──────────────────────┬──────────────────────────────────┐
│ 左侧菜单栏  │ 左侧配置栏（按路由显示） │ 主内容区                           │
│ AppMenu    │ analyze/create 显示 │ RouterView 默认内容                │
├────────────┼──────────────────────┼──────────────────────────────────┤
│ 首页        │ 拆解脚本配置          │ 结果页 / 首页 / 收藏 / 设置 / 个人中心 │
│ 拆解脚本    │ 脚本生成配置          │                                  │
│ 脚本生成    │                      │                                  │
│ 我的收藏    │                      │                                  │
│ 设置        │                      │                                  │
└────────────┴──────────────────────┴──────────────────────────────────┘
```

### 1.2 当前页面现状

| 页面 | 当前实现 | 现状结论 |
|------|----------|----------|
| 首页 | Hero + 两张功能卡片 + 即将推出 | 适合继续叠加公告和 SEO 内容区 |
| 拆解脚本 | 左侧上传/模型配置，右侧结果工作台 | 应保留现有工作流，只增加辅助入口 |
| 脚本生成 | 左侧单面板信息流，右侧结果工作台 | 当前已不是旧 Tabs 模式，后续设计需基于此扩展 |
| 我的收藏 | 顶部搜索 + 按日期分组的卡片列表 | 历史记录可参考其“搜索 + 分组/列表”的信息结构 |
| 设置 | 居中单列 API Key 表单 | v0.2.5 需要重构为配置中心 |
| 个人中心 | 居中卡片式信息展示 | 可作为反馈记录入口的后续承载点 |

### 1.3 设计原则

- 保留现有 `AppLayout` 壳层，不额外引入新的全局导航体系。
- 历史记录通过左侧菜单新增入口接入，而不是替换收藏页。
- 公告、SEO、设置重构都在现有页面基础上增量演进。
- 生成页继续沿用现有单面板信息流，不回退为 Tabs 设计。

## 3. 页面详细设计

### 3.1 首页（在现有首页基础上增强）

**当前现状**：
- 顶部是 `YouMedHub` Hero 文案。
- 中间是“拆解脚本 / 生成脚本”两张入口卡片。
- 底部是“即将推出”模块。

**v0.2.5 调整目标**：
- 在不推翻现有卡片结构的前提下，增加公告横幅和 SEO 静态内容。

**目标布局（ASCII）**：

```
┌──────────────────────────────────────────────────────────────┐
│  [公告横幅] v0.2.5 已上线：历史记录 / 英文支持 / 功能反馈 [查看] │
├──────────────────────────────────────────────────────────────┤
│                         Hero 区                               │
│                 YouMedHub / 简介 / 登录提示                    │
├──────────────────────────────────────────────────────────────┤
│                   两张现有功能入口卡片                         │
│                [拆解脚本]         [生成脚本]                   │
├──────────────────────────────────────────────────────────────┤
│                     即将推出模块（保留）                       │
├──────────────────────────────────────────────────────────────┤
│             新增 SEO 内容区：功能说明 / FAQ / 亮点             │
├──────────────────────────────────────────────────────────────┤
│                 页脚：更新公告 / 文档 / Sitemap                │
└──────────────────────────────────────────────────────────────┘
```

**交互说明**：
1. 有新公告时，Hero 上方展示轻量公告横幅。
2. 首页首次进入若有未读公告，则主内容渲染后再弹公告浮层。
3. SEO 内容区放在现有功能卡片与即将推出模块之后，保证不影响当前入口优先级。
4. 英文公开入口可作为按钮或轻量链接置于首页可见区域。

---

### 3.2 拆解脚本工作台（沿用现有结构）

**当前现状**：
- 左侧配置栏包含视频上传区和分析控制区。
- 右侧内容区顶部是结果工具栏，下方是思考面板和结果区。
- 未生成结果时，右侧只显示空状态。

**v0.2.5 调整目标**：
- 保持当前主流程不变，仅补充 API Key 引导、公告入口和语言适配能力。

**当前/目标结构（ASCII）**：

```
┌──────────────────────┬──────────────────────────────────────────┐
│ 左侧配置栏            │ 右侧结果区                               │
├──────────────────────┼──────────────────────────────────────────┤
│ 上传视频              │ 结果工具栏                               │
│ 模型选择              │ 思考面板                                 │
│ 思考模式              │ Markdown / 分镜表格                      │
│ 开始分析按钮          │ 无结果时为空状态                         │
│ 缺少 Key 提示         │                                          │
└──────────────────────┴──────────────────────────────────────────┘
```

**v0.2.5 增量点**：
- 缺少 API Key 时，提示区增加“去设置页配置”操作。
- 结果区工具栏可增加“写入历史记录成功”的轻提示，但不改动主体布局。
- 文案接入多语言后，保持现有按钮和区块顺序不变。

---

### 3.3 脚本生成工作台（基于现有单面板信息流演进）

**当前现状**：
- 左侧 `CreateModePanel` 已是单面板信息流：参考图片、视频类型、视频要求、参考脚本、目标时长、脚本数量、模型选择、思考模式。
- 生成成功后配置栏会自动收起，突出右侧结果区。

**设计结论**：
- v0.2.5 不再规划旧版“从零创作 / 参考生成 Tabs”结构。
- 历史记录和语言设置需要适配现有单面板表单流。

**当前/目标结构（ASCII）**：

```
┌──────────────────────┬──────────────────────────────────────────┐
│ 左侧生成配置栏        │ 右侧结果区                               │
├──────────────────────┼──────────────────────────────────────────┤
│ 参考图片              │ 结果工具栏                               │
│ 视频类型按钮组        │ 思考面板                                 │
│ 视频要求文本域        │ Markdown / 分镜表格                      │
│ 参考脚本选择          │ 多方案结果切换（当前已有数据能力）       │
│ 时长 / 数量           │ 无结果时为空状态                         │
│ 模型 / 思考 / 提交    │                                          │
└──────────────────────┴──────────────────────────────────────────┘
```

**v0.2.5 增量点**：
1. 缺少 API Key 时增加直达设置页入口。
2. 生成成功后自动写入历史记录，并支持从历史记录回填继续生成。
3. 多语言仅改文案和请求语言约束，不改变当前表单结构。

### 3.4 历史记录页（新增）

**设计来源**：
参考当前收藏页已有的“顶部搜索 + 卡片列表”结构，但强化筛选与详情能力。

**页面布局（ASCII）**：

```
┌──────────────────────────────────────────────────────────────┐
│ 历史记录                                           [搜索框]  │
├──────────────────────────────────────────────────────────────┤
│ [全部模式 ▼] [全部状态 ▼] [最近 30 天 ▼] [按时间倒序 ▼]      │
├──────────────────────────────┬───────────────────────────────┤
│ 左侧记录列表                  │ 右侧详情面板                  │
│ ┌──────────────────────────┐ │ 标题 / 模式 / 模型 / 时间     │
│ │ 从零创作  成功  今天15:32 │ │ 输入摘要 / 结果预览 / 分镜表  │
│ │ 30秒产品视频 / 24镜头     │ │ [继续生成] [收藏] [导出]      │
│ └──────────────────────────┘ │                               │
│ ┌──────────────────────────┐ │                               │
│ │ 视频拆解  失败  昨天      │ │                               │
│ │ API Key 无效             │ │                               │
│ └──────────────────────────┘ │                               │
└──────────────────────────────┴───────────────────────────────┘
```

**交互说明**：
- 默认按时间倒序。
- 桌面端使用“列表 + 详情”双栏；移动端退化为单列卡片流。
- 失败记录突出错误信息，并给出建议操作。
- 继续生成跳回对应页面并回填输入摘要。

### 3.5 设置页（从单列表单升级为配置中心）

**当前现状**：
- 居中单列 `max-w-md`，只有 API Key 输入框和保存按钮。

**v0.2.5 调整目标**：
- 重构为多分区配置中心，但仍保持与当前简洁卡片风格一致。

**目标布局（ASCII）**：

```
┌──────────────────────────────────────────────────────────────┐
│ 设置                                                         │
├──────────────────────────────────────────────────────────────┤
│ [语言设置卡片]                                               │
│ 界面语言 [简体中文 ▼]   AI 回复语言 [English ▼]              │
├──────────────────────────────────────────────────────────────┤
│ [API Key 引导卡片]                                           │
│ Step 1 前往百炼控制台 [立即前往]                              │
│ Step 2 创建 / 查看 Key                                       │
│ Step 3 粘贴保存 [sk-********] [校验并保存] [清空]             │
├──────────────────────────────────────────────────────────────┤
│ [功能反馈卡片]                                               │
│ 类型 / 标题 / 描述 / 联系方式 / 提交                          │
├──────────────────────────────────────────────────────────────┤
│ [最近反馈记录]（已登录用户）                                  │
└──────────────────────────────────────────────────────────────┘
```

**交互说明**：
1. 页面整体仍保持单页滚动，不强制做复杂后台式设置中心。
2. API Key 区域优先级最高，放在语言设置之后或同屏可见位置。
3. 已保存 Key 使用脱敏展示，支持修改和清空。
4. 反馈记录区可先做轻量列表，不阻塞反馈表单上线。
