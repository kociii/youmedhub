## 需求 1：存储方案功能验证与测试

### 1.1 背景与目标

**当前问题**：
v0.2.3 规划的存储方案重构功能（dashscope-upload.ts、useVideoAnalysis.ts 更新等）已基本实现，但缺乏系统性的功能验证和测试覆盖。

**目标**：
- 验证视频上传全流程（选择文件 → 生成本地预览 → 提交时上传 → 获取 oss:// URL）
- 验证图片上传全流程（参考生成模式）
- 验证 AI API 调用包含正确的 Header
- 验证模型切换时文件重新上传机制

### 1.2 需求详情

**功能要求**：
1. 视频文件上传验证
   - 支持 MP4/MOV/AVI 格式
   - 文件大小限制 100MB
   - 上传进度显示正常
   - 返回 `oss://` 格式 URL

2. 图片文件上传验证（参考生成模式）
   - 支持 JPEG/PNG/WebP/GIF 格式
   - 文件大小限制 20MB
   - 支持多张图片同时上传

3. API 调用验证
   - `X-DashScope-OssResourceResolve: enable` Header 正确添加
   - `oss://` URL 被正确解析
   - 思考模式参数正常传递

4. 模型切换验证
   - 切换模型后 videoUrl 清空
   - 切换模型后 imageUrls 清空
   - 上传状态重置为 idle

**非功能要求**：
- 上传失败时错误提示清晰
- 网络中断后支持重试
- 凭证过期后自动重新获取

### 1.3 验收标准

- [x] 视频文件上传成功并返回 `oss://` URL
- [x] 图片文件上传成功并返回 `oss://` URL
- [x] 所有 AI API 调用包含 `X-DashScope-OssResourceResolve` Header
- [ ] 模型切换后正确清空已上传的 URL
- [x] 上传进度显示准确
- [x] 错误处理机制正常工作

## 需求 2：上传流程端到端测试

### 2.1 背景与目标

**当前问题**：
新的上传流程（选择文件时不上传，提交时才上传）需要完整的端到端测试验证。

**目标**：
- 覆盖视频分析页面的完整流程
- 覆盖脚本生成页面的完整流程
- 覆盖参考生成模式的图片上传流程

### 2.2 需求详情

**测试场景**：

1. 视频分析流程
   - 选择本地视频 → 生成本地预览 → 点击分析 → 上传文件 → AI 分析 → 显示结果

2. 从零创作流程
   - 选择本地视频 → 点击生成 → 上传文件 → AI 生成 → 显示脚本

3. 参考生成流程
   - 选择本地视频 → 添加参考图片 → 点击生成 → 上传视频和图片 → AI 生成 → 显示脚本

4. 模型切换场景
   - 上传视频 → 切换模型 → 验证 URL 清空 → 重新提交 → 重新上传

### 2.3 验收标准

- [x] 视频分析流程端到端测试通过
- [ ] 从零创作流程端到端测试通过
- [ ] 参考生成流程端到端测试通过
- [ ] 模型切换后重新上传机制正常
