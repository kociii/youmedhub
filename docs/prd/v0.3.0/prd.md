# v0.3.0 项目模块与全屏项目画布（MVP）

## 版本概述

v0.3.0 不再把画布定义为零散的编辑页，而是定义为一个独立的功能板块：**项目**。  
项目包含两部分：

1. **项目管理页**：查看、搜索、手动新建项目
2. **项目画布页**：在浏览器新标签页中全屏打开，进行脚本卡片组织

用户可以从“脚本拆解”和“脚本创作”进入项目：**每次点击都新建一个项目**，并把当前脚本带入项目画布，生成一张或多张脚本卡片。与此同时，项目也必须支持从项目管理页手动创建空白项目。

### 产品定位（明确）

项目画布的定位是：**AI 创作工具的编排中枢**，不是普通白板。  
它用于将文本、脚本、图片、视频、AI 调用等卡片节点进行可视化编排，形成可复用的内容生产方案。

核心价值：

1. 把分散的素材和 AI 过程组织成可执行的创作链路。
2. 通过节点化编排提高复用效率，而不是每次从零开始。
3. 支持将“输入-处理-输出”流程沉淀为模板化项目。

## 本版本增量调整

### 当前已实现结构

- 分析/创作结果存在统一结果区和工具栏。
- 脚本结果已有结构化字段。

### 本版本增量调整

1. 在左侧菜单中新增“项目”入口。
2. 新增项目管理页 `/projects`。
3. 新增全屏项目画布页 `/projects/:id`。
4. 从分析页/创作页点击“新建项目”，会：
   - 先创建一个新的项目记录
   - 再把当前脚本写为初始画布快照
   - 最后以新标签页打开该项目
5. 从项目管理页可以手动创建空白项目。

## 关键用户流程

### 流程 1：从脚本拆解进入项目

1. 用户在 `/analyze` 完成脚本拆解。
2. 在结果工具栏点击“新建项目”。
3. 系统创建一个新的项目记录，来源标记为 `analyze`。
4. 当前脚本被写入项目首个快照，并生成脚本卡片。
5. 浏览器以新标签页打开 `/projects/:id` 全屏项目画布。

### 流程 2：从脚本创作进入项目

1. 用户在 `/create` 生成一个候选脚本。
2. 选定当前方案。
3. 点击“新建项目”。
4. 系统创建项目并把当前方案写入项目初始画布。
5. 新标签页打开项目画布。

### 流程 3：手动新建项目

1. 用户进入 `/projects`。
2. 点击“新建项目”。
3. 系统创建空白项目。
4. 新标签页打开 `/projects/:id`。

## 功能需求

### F1. 项目管理页

- 路由：`/projects`
- 作用：
  - 展示项目列表
  - 支持搜索
  - 支持手动新建项目
  - 支持打开已有项目

### F2. 全屏项目画布页

- 路由：`/projects/:id`
- 必须在新标签页中全屏展示。
- 不复用当前 `AppLayout` 三栏壳层。

### F3. 从拆解/创作结果新建项目

- 入口放在 `ResultToolbar`。
- 每次点击都创建 **新的项目**，不是复用旧项目。
- 当前脚本导入为一个或多个 `script` 卡片节点。

### F4. 手动新建空白项目

- 用户可以不依赖脚本结果，直接在项目管理页新建空白项目。
- 空白项目默认只创建项目主记录与初始空快照。

### F5. 项目画布基础交互

- 平移、缩放、重置视图
- 单击选中节点
- 拖拽移动节点
- 快速创建 `note` 卡片

### F6. Supabase 数据落库

- 项目主记录写入 Supabase
- 项目快照写入 Supabase
- 首版先保证“项目可创建、可打开、可继续编辑”

## 非目标（后置版本）

以下能力不放入 v0.3.0：

- 节点连线
- 项目内图片/视频 AI 生成
- 导出 PNG / JSON / Excel
- 富媒体节点完整体系
- 云端协同

这些能力拆到 `v0.3.1`、`v0.3.2`。
