# v0.3.0 开发计划

> 项目模块与全屏项目画布（MVP）

---

## 版本状态

```
v0.3.0 ⏳ 规划中
```

**版本目标**：
- 建立独立的“项目”功能板块
- 增加项目管理页 `/projects`
- 增加全屏项目画布页 `/projects/:id`
- 支持从脚本拆解/脚本创作一键新建项目
- 支持手动创建空白项目

---

## 已完成审查

### 已审查文件

- `src/App.vue`
- `src/router/index.ts`
- `src/components/layout/AppLayout.vue`
- `src/components/layout/AppMenu.vue`
- `src/views/AnalyzePage.vue`
- `src/views/CreatePage.vue`
- `src/components/RightPanel.vue`
- `src/components/ResultToolbar.vue`
- `src/components/ScriptTable.vue`
- `src/composables/useVideoAnalysis.ts`

### 审查结论

- 当前结果区已有成熟入口位置，可以直接承载“新建项目”动作。
- 如果项目要独立成板块，就必须同时有“项目管理”和“项目画布”，不能只做一个画布页。
- 由于项目需要被列表化管理，v0.3.0 就应该有 Supabase 项目表，而不是把数据全部后移。

## 里程碑

- **M1**：主应用出现“项目”入口和项目列表页
- **M2**：可以手动新建空白项目
- **M3**：可以从脚本拆解/脚本创作一键新建项目并打开项目画布

---

## 关键原则

- v0.3.0 的核心不是“炫技画布”，而是“项目闭环”。
- 任何入口都围绕“创建一个新项目”展开，不复用旧项目。
- 项目画布保持新标签页全屏体验，项目管理页留在主应用壳层。
