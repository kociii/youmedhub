### 基线结论

- 主应用仍是 Web 三栏壳层。
- 项目画布继续保持“新标签页 + 全屏编辑”的总体方向。
- `v0.3.1` 不回退到壳层内嵌画布，而是在全屏项目编辑器中增加更多编辑能力。
- 当前项目画布已经是 DOM 卡片方案，图片 / 视频节点仍主要是占位型结构。
- 当前尚未建立资源库、拖拽上传、上传前视频校验和边缘缩放的统一规则。

## 本版本增量

- 支持从 `script` 节点扩展到 `text`、`image`、`video` 节点。
- 增加右侧属性面板，编辑尺寸、标题、描述等基础属性。
- 增加撤销/重做。
- 增强项目列表页，便于搜索、分组和重新打开最近项目。
- `image` / `video` 节点涉及的文件存储统一走阿里云 OSS，本地项目只保存 OSS 对象信息与展示元数据，不保存浏览器 `blob:` 地址。
- 图片 / 视频节点点击后弹出媒体操作浮层，支持：
  - 本地上传
  - 从资源库选择
- 支持从本地将图片 / 视频直接拖入画布，拖入后先上传，再创建或填充节点。
- 视频上传前强制校验：
  - 时长 ≤ 5 分钟
  - 文件大小 ≤ 100MB
- 所有卡片节点支持拖拽边缘缩放。

### 节点体系（v0.3.1 定义）

v0.3.1 将项目节点体系定义为“可扩展卡片”：

1. `script`：脚本卡片（来自拆解/创作结果）
2. `text`：文本卡片（自由文案、备注、提示词片段）
3. `image`：图片卡片（OSS 资源，图片充满卡片、按原比例展示）
4. `video`：视频卡片（OSS 资源，视频充满卡片、按原比例展示）
5. `note`：轻量便签卡片
6. `ai_call`：AI 调用卡片（v0.3.2 实做）

说明：`ai_call` 在 v0.3.1 先完成类型预留，具体交互在 v0.3.2 落地；`video` 在本版纳入正式媒体卡片范围。
